# 后续可优化方向

以下为项目在功能、体验与工程化上可继续演进的方向，按主题归类，便于排期。

---

## 产品与体验

- **草稿持久化**：浏览器 `localStorage` 保存简历、JD 与最近一次分析摘要，刷新不丢稿（注意敏感信息提示）。
- **流式 UI**：显示首字节耗时、可取消进行中的 SSE 请求（`AbortController`）。
- **双语界面**：恢复或扩展 i18n（当前默认中文 UI）。
- **无障碍**：键盘焦点顺序、`aria-live` 对流式区域、错误提示与表单标签强化。
- **模板改写 UX**：在模板区增加“版本历史与回滚”（V1/V2/V3），避免重写后回不去。
- **模板对比视图**：预览页支持“上一版 vs 当前版”并排对比，降低人工复查成本。

## 模型与业务效果

- **分岗位策略**：按 JD 行业（研发/产品/运营）切换轻量 system 提示或 few-shot，提高建议可执行性。
- **事实校验提示**：对用户粘贴的量化数据打「需自行核实」标签，减少过度信任模型改写。
- **多版本对比**：同一轮保存多版优化稿 diff 视图，便于选稿。
- **模板返修记忆持久化**：目前记忆在进程内，后续可落地到 Redis/DB，支持重启后继续返修。

## 性能与成本

- **精确 Token**：接入官方 tokenizer 或近似库，替代字符启发式估算。
- **分布式缓存**：多副本部署时用 **Redis** 共享「简历+JD」分析缓存；流式与非流式共用缓存键策略。
- **连接池与重试**：对 DeepSeek HTTP 客户端统一超时、退避重试与熔断（避免雪崩）。

## 可靠性与安全

- **鉴权**：可选 API Key、JWT 或内网网关，避免公网裸奔。
- **审计**：结构化日志字段（不含简历正文）、与 `X-Request-ID` 对齐的查询界面。
- **上传安全**：病毒扫描、MIME 校验、更严格的文件大小与页数限制。

## 测试与质量

- **契约测试**：OpenAPI schema 与前端类型生成（减少 SSE 字段漂移）。
- **Playwright**：关键路径 E2E（上传 → 分析 → 优化 → 导出），对真实 API 可设为可选 job。
- **CI 流水线**（可选）：在自有 Git 平台配置定时或 PR 触发，执行 `pip install`、`pytest`、`pip-audit` 等（仓库未内置工作流文件，按需自行添加）。
- **模板链路专项测试**：覆盖“同模板返修记忆”“切模板清空上下文”“多页预览完整返回”“错误解释可读性”。

## 可观测与运维

- **OpenTelemetry**：HTTP 客户端与路由级 trace；导出到 Jaeger/OTLP。
- **健康检查扩展**：依赖 DeepSeek 的浅探测（慎用，避免刷配额）。
- **阶段耗时监控**：记录模板改写各阶段耗时（LLM/编译/打包），用于优化进度条与容量评估。

## 前端工程

- **统一构建路径**：以 `frontend/` Vite 为主，CDN 版作为 fallback 文档化；或仅保留一种部署方式降低维护成本。
